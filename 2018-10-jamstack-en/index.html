What is JAMStack?

"A modern web development architecture based on client-side JavaScript, reusable APIs, and prebuilt Markup."

Mathias Biilman, CEO of Netlify

The three pillars

  • JavaScript (code)
  • API's (content)
  • Markup (templates)

This results in a static site

Static sites are:

  • Fast
    Sites prebuilt into simple, static files
  • Safe
    No direct requests to a server/database by the user
  • Scalable
    Hosting for static files is simple to setup and very scalable

What does JAMStack solve in these situations?

  • No overhead running a CMS / framework
  • Reduced complexity of hosting and server
  • No laggyness caused by heavy APIs/CMSes
  • Improved security of static files
  • More freedom with decoupled front-ends
  • Files worden gegenereerd bij deployment
  • Elke hosting die een bak bestanden kan serveren is voldoende
  • Bezoeker heeft geen last van externe services omdat deze aangesproken worden bij deployment
  • Omdat er geen CMS of framework bij een request aan te pas komt, zijn er ook veel minder/geen aanvalshoeken

But wait, there's more!

This stuff gets way easier:

Global CDN, Caching, Cache invalidation,
Atomic deploys, Incremental builds, Rollbacks,
Continuous deployment, Command line deployment,
Critical rendering, Asset optimization, etc.

What is this very suited for?

  • Sites with a lot of content
  • Content from existing APIs
  • Really high performance requirements

What is it less suited for?

  • Real-time / user-specific data
  • Webshops
  • Form handling
  • Search

But not impossible...

Solutions

  • Single Page Apps
    React, Angular, Vue
  • Payment & Shopping cart services
    Shopify, Stripe
  • Form services
    Wufoo, TypeForm, Netlify Forms
  • Search services
    Google Search, Amazon Search, Algolia
  • Authentication services
    Auth0, OAuth, Netlify Identity

So how would we realize this?

  • Set up a CMS / framework with an API
  • Static site generator that consumes that API
  • Hosting for simple static files
  • Deployment which builds the site
  • Global CDN for superfast worldwide availability
